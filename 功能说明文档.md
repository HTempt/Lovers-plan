# 《双人岛》微信小程序 — 功能说明文档

**版本**：V1.0  
**产品类型**：微信小程序  
**目标用户**：18-35 岁情侣

---

## 一、产品概述

### 1.1 产品定位

帮助情侣记录恋爱过程、规划共同目标、共享日常状态的双人协作型成长产品。不提供聊天功能，聊天依赖微信生态。

### 1.2 产品专注方向

| 方向 | 说明 |
|------|------|
| 回忆记录 | 保存恋爱中的重要时刻 |
| 双人成长 | 一起完成任务、培养共同习惯 |
| 目标规划 | 共同制定愿望和目标 |
| 纪念提醒 | 不错过重要日子 |
| 日常陪伴 | 了解彼此当下的状态 |

### 1.3 核心价值

- **记录** — 保存恋爱过程中的重要时刻
- **规划** — 共同制定未来目标
- **陪伴** — 了解彼此状态
- **成长** — 一起完成任务

---

## 二、业务流程

```
微信登录 → 创建/加入情侣 → 绑定成功 → 进入首页
```

- 首次使用：微信授权登录 → 创建情侣档案 → 生成6位邀请码 → 另一半输入邀请码绑定
- 一个账号仅能绑定一个对象
- 解除绑定后历史数据保留，关系解除

## 四、功能详情

### 4.1 情侣绑定模块

- **创建邀请**：用户点击"邀请另一半"，系统生成6位邀请码，有效期24小时
- **接受邀请**：输入邀请码完成绑定
- **规则**：一个账号仅绑定一个对象；解除绑定后历史数据保留

### 4.2 首页模块

| 区域 | 内容 |
|------|------|
| 顶部 Hero | 恋爱 / 结婚天数（"我们恋爱啦/我们结婚啦" + 天数 + 引号文案） |
| 今日状态 | 双人卡片：左侧"我"可点击进入状态设置页，右侧对方仅展示不可点击；显示状态名称、心情、持续时长 |
| 岛屿动态 | 按时间倒序聚合展示情侣的各类动态，默认10条，支持上滑分页加载，点击跳转对应详情页 |

**状态设置页**（保持不变）：
- 预置模板：工作中、学习中、睡觉中、运动中、游戏中、路上
- 心情标签：开心、平静、难过、生气
- 保存/清空操作
- 状态持续超过24小时自动过期清理

**岛屿动态类型**：

| 动态类型 | 触发时机 | 图标 | 点击跳转 |
|----------|----------|------|----------|
| 日记发布 | 创建日记后自动记录 | 📝 | 日记页 |
| 状态变更 | 更新状态后自动记录 | 🟢 | 状态设置页 |
| 任务完成 | 打卡确认达成目标时 | 🎯 | 任务页 |
| 愿望达成 | 进度达到目标或手动标记 | ✨ | 愿望页 |
| 纪念日新增 | 创建自定义或系统纪念日 | 💝 | 纪念日页 |

> 动态数据从各业务模块自动采集，无需手动发布。首页不再展示独立的功能卡片（提醒/回忆/愿望），统一由动态流承载。

### 4.3 恋爱日记模块

| 视图 | 说明 |
|------|------|
| 时间轴 | 按创建时间倒序，展示标题、内容、媒体缩略图、地点，日期格式 `yyyy-MM-dd HH:mm:ss` |
| 相册 | 按月聚合，仅含媒体的日记参与展示 |
| 地图足迹 | 根据日记地点自动生成足迹分布 |

**创建日记**：
- 字段：标题、内容、图片/视频（最多9张）、地点
- 删除规则：仅创建者可删除，删除后进入回收站保留30天

### 4.4 共享状态模块

- 预置模板：工作中、学习中、睡觉中、运动中、游戏中、路上
- 可选心情标签：开心、平静、难过、生气
- 展示信息：状态名称、心情、开始时间、持续时长
- 双方状态在首页双人卡片中并排展示，中间显示 💗 连接
- 用户设置状态后，自动通过微信订阅消息通知另一半

### 4.5 共享待办模块

- 字段：标题、截止时间、执行人、优先级（高/中/低）
- 状态：待完成 / 已完成 / 已过期
- 提醒规则：截止前1小时通过微信订阅消息提醒执行人
- 支持每天/每周/每月重复

### 4.6 情侣任务模块

- 字段：标题、目标次数、截止时间
- 打卡流程：A打卡 → B确认 → 完成1次
- 状态：进行中 / 已完成 / 已过期
- 成就徽章：连续7天打卡、旅行达人、健身达人等
- 待确认的任务通过微信订阅消息提醒对方确认

### 4.7 愿望清单模块

- 字段：标题、分类（旅行/生活/成长）、目标金额、当前金额、目标日期
- 进度公式：`当前金额 / 目标金额 × 100%`
- 完成状态：目标达成 → "梦想成真" → 归档
- 目标日期临近时通过微信订阅消息提醒

### 4.8 纪念日模块

- **系统纪念日**：恋爱纪念日（绑定成功后自动生成）
- **自定义纪念日**：用户自行创建（第一次见面、旅行纪念等）
- **排序**：按纪念日日期倒序展示（最新的在前）
- **提醒设置**：提前1/3/7/15天，通过微信订阅消息在每天8:00检查推送
- **展示信息**：标题、日期、农历日期、已持续天数、距离下次天数

### 4.9 个人中心模块

| 功能 | 说明 |
|------|------|
| 用户信息 | 昵称修改、头像更换（相册/拍照）、性别设置、手机号绑定 |
| 情侣信息 | 展示另一半头像和昵称 |
| 恋爱报告 | 统计数据总览（日记/照片/纪念日/任务/待办/愿望等） |
| 消息订阅 | 管理5种订阅：状态通知、纪念日提醒、待办提醒、任务确认、愿望提醒 |
| 解除绑定 | 保留数据、解除关系 |
| 退出登录 | 清除登录状态 |

### 4.10 消息订阅模块

后端已完整实现微信订阅消息推送能力：

| 订阅类型 | 推送时机 | 推送频率 |
|----------|----------|----------|
| 🔔 对方状态通知 | 对方设置状态时立即推送 | 每次设置 |
| 💝 纪念日提醒 | 达到提前提醒天数时（每天8:00） | 每天最多1条 |
| ✅ 待办到期提醒 | 截止前1小时（每小时检查） | 每条待办1次 |
| 🎯 任务确认提醒 | 对方打卡后待确认（每天9:00） | 每天最多1条 |
| ✨ 愿望截止提醒 | 目标到期的前7天起（每天10:00） | 每天最多1条 |

> ⚠️ 需要在微信公众平台申请模板消息模板，并将真实模板 ID 填入 `application.yml`

## 五、技术架构

```
微信小程序
    ↓
Spring Boot (API Gateway)
    ↓
Redis (缓存 / Access Token)
    ↓
MySQL (持久化)
    ↓
MinIO (对象存储)
```

### 5.1 数据库表

| 表名 | 说明 |
|------|------|
| `user` | 用户：openid, 昵称, 头像, 性别 |
| `couple` | 情侣关系：user_a, user_b, love_date |
| `diary` | 日记：标题, 内容, 地点 |
| `diary_media` | 日记媒体：图片/视频 |
| `todo` | 待办：标题, 截止时间, 执行人, 优先级 |
| `task` | 任务：标题, 目标次数, 当前次数 |
| `task_record` | 打卡记录：用户, 状态 |
| `wish` | 愿望：标题, 分类, 金额, 目标日期 |
| `anniversary` | 纪念日：标题, 日期, 提前提醒天数 |
| `user_status` | 状态：名称, 心情, 持续时间 |
| `activity` | 岛屿动态：类型, 标题, 描述, 关联ID |

### 5.2 API 设计

**统一返回格式**：
```json
{ "code": 200, "message": "success", "data": {} }
```

**认证方式**：JWT Token，请求头 `Authorization: Bearer xxx`

**模块路由**：

| 模块 | 路由 |
|------|------|
| 认证 | `/api/auth/*` |
| 情侣 | `/api/couple/*` |
| 首页 | `/api/home/*` |
| 日记 | `/api/diary/*` |
| 任务 | `/api/task/*` |
| 待办 | `/api/todo/*` |
| 愿望 | `/api/wish/*` |
| 纪念日 | `/api/anniversary/*` |
| 状态 | `/api/status/*` |
| 统计 | `/api/statistics/*` |
| 订阅 | `/api/subscribe/*` |
| 动态 | `/api/activity/*` |
| 通用 | `/api/common/*` |
