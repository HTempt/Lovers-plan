# 《双人岛》微信小程序 — 功能说明文档

**版本**：V1.0  
**产品类型**：微信小程序  
**目标用户**：18-35 岁情侣

---

## 一、产品概述

### 1.1 产品定位

帮助情侣记录恋爱过程、规划共同目标、共享日常状态的双人协作型成长产品。不提供聊天功能，聊天依赖微信生态。

### 1.2 产品专注方向

| 方向 | 说明 |
|------|------|
| 回忆记录 | 保存恋爱中的重要时刻 |
| 双人成长 | 一起完成任务、培养共同习惯 |
| 目标规划 | 共同制定愿望和目标 |
| 纪念提醒 | 不错过重要日子 |
| 日常陪伴 | 了解彼此当下的状态 |

### 1.3 核心价值

- **记录** — 保存恋爱过程中的重要时刻
- **规划** — 共同制定未来目标
- **陪伴** — 了解彼此状态
- **成长** — 一起完成任务

---

## 二、业务流程

```
微信登录 → 创建/加入情侣 → 绑定成功 → 进入首页
```

- 首次使用：微信授权登录 → 创建情侣档案 → 生成6位邀请码 → 另一半输入邀请码绑定
- 一个账号仅能绑定一个对象
- 解除绑定后历史数据保留，关系解除

## 四、功能详情

### 4.1 情侣绑定模块

- **创建邀请**：用户点击"邀请另一半"，系统生成6位邀请码，有效期24小时
- **接受邀请**：输入邀请码完成绑定
- **规则**：一个账号仅绑定一个对象；解除绑定后历史数据保留

### 4.2 首页模块

#### 页面布局（自上而下）

| 区域 | 内容 |
|------|------|
| 顶部 Hero | 恋爱 / 结婚天数（"我们恋爱啦/我们结婚啦" + 天数 + 引号文案） |
| 双人状态卡片 | 并排展示：左侧"我"可点击进入状态设置页，右侧对方仅展示不可点击；中间 💗 连接动画；显示状态名称、心情、持续时长 |
| 双列卡片行 | 左侧智能卡片 + 右侧爱情树卡片（固定），左右等宽 |
| 💌 今日提醒 | 双列宫格展示各模块提醒卡片（纪念日/任务/愿望/胶囊），按优先级排序 |
| 🌊 岛屿动态 | 按时间倒序聚合展示情侣的各类动态，默认10条，支持上滑分页加载 |

#### 左侧卡片智能切换

根据当前数据状态自动切换展示内容，优先级如下：

| 优先级 | 类型 | 触发条件 | 展示内容 |
|--------|------|----------|----------|
| 1 | 🕰 时光机 | 有历史同日回忆 | 缩略图/emoji + 标题 + 日期 |
| 2 | 🌏 情侣足迹 | 有已到达城市 | 城市标签列表（最多3个） |
| 3 | 🎯 愿望进度 | 有进行中的愿望 | 进度条 + 百分比 |
| 4 | ❤️ 恋爱里程碑 | 兜底展示 | 距离下个纪念日天数 |

#### 爱情树卡片（右侧固定）

展示爱情树的精简状态：

```
┌─────────────────┐
│ 🌱 爱情树       │
│ 🌰 Lv2 幼苗     │  ← 等级图标 + Lv + 等级名
│ ▓▓▓▓░░ 200/500  │  ← 绿色进度条 + 成长值/阈值
│ 每日访问自动签到 🌱│  ← 签到状态（已签到显示 ✓）
└─────────────────┘
```

- 点击卡片跳转爱情树详情页
- 已签到显示：✅ 已签到 + "今日签到 +2 · 连续 N 天"
- 未签到：显示 "每日访问自动签到 🌱"

#### 今日提醒

按优先级排序展示：

| 类型 | 图标 | 触发条件 |
|------|------|----------|
| 纪念日 | 💝 | 距离纪念日 ≤ 提前提醒天数 |
| 任务 | 🎯 | 有待确认的打卡任务 |
| 愿望 | ✨ | 愿望目标日期临近 |
| 胶囊 | 💌 | 胶囊已到期可开启 |

#### 岛屿动态类型

| 动态类型 | 触发时机 | 图标 | 点击跳转 |
|----------|----------|------|----------|
| 日记发布 | 创建日记后自动记录 | 📝 | 日记页 |
| 状态变更 | 更新状态后自动记录 | 🟢 | 状态设置页 |
| 任务完成 | 打卡确认达成目标时 | 🎯 | 任务页 |
| 愿望达成 | 进度达到目标或手动标记 | ✨ | 愿望页 |
| 纪念日新增 | 创建自定义或系统纪念日 | 💝 | 纪念日页 |
| 胶囊封存 | 创建/写入胶囊后自动记录 | 💌 | 胶囊列表 |
| 胶囊开启 | 点击开启胶囊后自动记录 | 💌 | 胶囊列表 |
| 成就解锁 | 达成成就条件后自动记录 | 🏆 | 成就页 |
| 爱情树升级 | 爱情树升级时自动记录 | 🌱 | 爱情树页 |

> 动态数据从各业务模块自动采集，无需手动发布。首页统一由动态流承载全部事件提醒。

#### 状态设置页

- 预置模板：工作中、学习中、睡觉中、运动中、游戏中、路上
- 心情标签：开心、平静、难过、生气
- 保存/清空操作
- 状态持续超过24小时自动过期清理

### 4.3 恋爱日记模块

| 视图 | 说明 |
|------|------|
| 时间轴 | 按创建时间倒序，展示标题、内容、媒体缩略图、地点，日期格式 `yyyy-MM-dd HH:mm:ss` |
| 相册 | 按月聚合，仅含媒体的日记参与展示 |
| 地图足迹 | 根据日记地点自动生成足迹分布 |

**创建日记**：
- 字段：标题、内容、图片/视频（最多9张）、地点
- 删除规则：仅创建者可删除，删除后进入回收站保留30天

### 4.4 共享状态模块

- 预置模板：工作中、学习中、睡觉中、运动中、游戏中、路上
- 可选心情标签：开心、平静、难过、生气
- 展示信息：状态名称、心情、开始时间、持续时长
- 双方状态在首页双人卡片中并排展示，中间显示 💗 连接
- 用户设置状态后，自动通过微信订阅消息通知另一半

### 4.5 共享待办模块

- 字段：标题、截止时间、执行人、优先级（高/中/低）
- 状态：待完成 / 已完成 / 已过期
- 提醒规则：截止前1小时通过微信订阅消息提醒执行人
- 支持每天/每周/每月重复

### 4.6 情侣任务模块

- 字段：标题、目标次数、截止时间
- 打卡流程：A打卡 → B确认 → 完成1次
- 状态：进行中 / 已完成 / 已过期
- 成就徽章：连续7天打卡、旅行达人、健身达人等
- 待确认的任务通过微信订阅消息提醒对方确认

### 4.7 愿望清单模块

- 字段：标题、分类（旅行/生活/成长）、目标金额、当前金额、目标日期
- 进度公式：`当前金额 / 目标金额 × 100%`
- 完成状态：目标达成 → "梦想成真" → 归档
- 目标日期临近时通过微信订阅消息提醒

### 4.8 纪念日模块

- **系统纪念日**：恋爱纪念日（绑定成功后自动生成）
- **自定义纪念日**：用户自行创建（第一次见面、旅行纪念等）
- **排序**：按纪念日日期倒序展示（最新的在前）
- **提醒设置**：提前1/3/7/15天，通过微信订阅消息在每天8:00检查推送
- **展示信息**：标题、日期、农历日期、已持续天数、距离下次天数

### 4.9 双人问答模块

#### 功能概述

每天生成1道情侣互动问题，双方分别作答，提升情侣互动频率与默契感。

#### 问题类型

| 类型 | 说明 | 示例 |
|------|------|------|
| 💭 回忆类 | 关于共同经历的问题 | 第一次见面在哪里？ |
| 💖 偏好类 | 了解彼此喜好 | TA最喜欢吃什么？ |
| 💕 情感类 | 感情相关问题 | 吵架后谁先道歉？ |
| 🌟 未来类 | 未来规划问题 | 希望几年后结婚？ |
| 😄 娱乐类 | 轻松趣味题 | 对方像什么动物？ |

#### 数据表

| 表名 | 说明 |
|------|------|
| `question` | 问题池（预置25+道题） |
| `daily_question` | 每日每题情侣的答题记录 |
| `daily_answer` | 双方答案记录 |

#### 答题流程

1. 用户进入答题页 → 系统自动分配今日问题
2. 展示4个选项（A/B/C/D），选择后提交
3. 双方都提交后 → 计算默契度
4. 展示结果页

#### 默契度规则

| 情况 | 分值 |
|------|------|
| 双方选择一致 | 💞 100% |
| 双方选择不同 | 💔 0% |

#### 奖励机制

| 行为 | 爱情树成长值 |
|------|-------------|
| 完成答题 | +2 |
| 默契一致 | +5（额外） |

#### 首页入口

今日提醒模块新增卡片「❤️ 今日问答」，显示当前状态：

- 未作答：今日问题待作答
- 一方已答：⏳ 等待TA作答
- 双方已答：✅ 双方已作答

点击卡片跳转答题页。

#### 岛屿动态联动

双方作答完成后自动记录：💞 完成一次双人问答

### 4.10 时光机模块

#### 功能概述

自动发现历史同月同日发生的事件，带你「回到过去的今天」。

#### 工作原理

| 当前日期 | 系统自动查询 |
|----------|--------------|
| 2026-06-19 | 2025-06-19、2024-06-19、2023-06-19 当天发生的内容 |

#### 内容来源与优先级

| 优先级 | 来源 | 示例 |
|--------|------|------|
| 1 | 带图片的日记 | 📝 一起吃火锅 🖼️ |
| 2 | 普通日记 | 📝 今天去看了电影 |
| 3 | 纪念日 | ❤️ 一周年纪念 |
| 4 | 愿望达成 | ✨ 一起去日本旅行 ✓ |
| 5 | 任务完成 | 🎯 健身打卡 100 次 ✓ |

#### 首页展示规则

- 采用双列卡片布局：**🕰 时光机**（左）+ **🌱 爱情树**（右）
- 首页仅展示**最近一条**回忆，避免页面过长
- 点击卡片跳转对应详情页查看全部
- 若当天无精确匹配，自动降级为 **±3 天** 范围匹配
- 无匹配时隐藏模块

#### 展示格式

```
┌─────────────────┐
│ 🕰 时光机       │
│ 回到过去的今天  │
│ ┌───┐           │
│ │ 🖼️│ 一起吃火锅 │
│ └───┘ 2025-06-19│
└─────────────────┘
```

- 有图片：左侧 100rpx 缩略图 + 右侧标题+日期
- 无图片：左侧 emoji 图标 + 右侧标题+日期
- 内容最多两行，超出省略号

#### 岛屿动态联动

新增动态类型：🕰 时光机发现了一段回忆

### 4.11 时光胶囊模块

#### 功能概述

封存当下的爱意与回忆，在未来某个时间重新开启。支持单人/双人两种模式。

#### 胶囊类型

| 类型 | 说明 |
|------|------|
| 给未来的 TA | 封存想对对方说的话 |
| 给未来的我们 | 写给双方的未来 |
| 生日胶囊 | 为对方生日准备的惊喜 |
| 纪念日胶囊 | 纪念日专属封存 |
| 愿望达成胶囊 | 记录达成愿望的时刻 |

#### 状态流转

```
DRAFT（草稿）→ SEALED（已封存）→ OPENABLE（可开启）→ OPENED（已开启）
```

| 状态 | 说明 | 内容可见 |
|------|------|----------|
| DRAFT | 双人模式等待对方写入 | 仅创建者可见 |
| SEALED | 已封存，不可查看内容 | ❌ 隐藏 |
| OPENABLE | 已到期，点击开启 | ❌ 隐藏（需先开启） |
| OPENED | 已开启，内容完全展示 | ✅ 可见 |

| 封存时长 | 说明 |
|----------|------|
| 30 天后 | 短期封存 |
| 90 天后 | 季度封存 |
| 180 天后 | 半年封存 |
| 365 天后 | 一年封存 |
| 自定义日期 | 自由选择日期 |

#### 支持内容

- 📝 文字
- 🖼️ 图片（最多9张）
- 🎤 语音
- 🎬 视频

#### 双人模式

双方分别写入内容，全部完成后自动封存：

1. **发起方** 创建时开启双人模式 → 胶囊以 DRAFT 状态保存
2. **伴侣** 在列表「待写入」标签页看到蓝色高亮胶囊 → 点击进入写入页
3. **伴侣** 填写内容+附件 → 点击「写入并封存」
4. 双方胶囊同时变更为 SEALED

#### 联动功能

| 联动 | 触发 | 说明 |
|------|------|------|
| 🌱 爱情树成长值 | 创建胶囊 +10 | `GROWTH_CAPSULE_CREATE` |
| 🌱 爱情树成长值 | 开启胶囊 +20 | `GROWTH_CAPSULE_OPEN` |
| 🌊 岛屿动态 | 封存胶囊 | 💌 你们封存了一颗时光胶囊 |
| 🌊 岛屿动态 | 开启胶囊 | 💌 一颗时光胶囊被开启 |
| 📬 微信订阅消息 | 胶囊到期 | 「你的时光胶囊已到期 ❤️」 |
| 🕰 时光机 | 发现历史同日回忆时 | 时光机「一年前的今天」联动 |

#### 开启流程

1. 到期后列表显示「📬 时光胶囊已成熟」
2. 点击弹出信封开启弹窗（信封弹跳动画）
3. 点击「开启胶囊」→ 展示创建日期、封存时长、内容详情

#### 解锁机制

采用**拉取式检查**：用户进入成就页面时，前端调用 `POST /achievement/check` 主动检查指定分类下所有未解锁的成就条件是否满足。条件满足时自动解锁并弹出提示。

解锁触发：
- 进入成就页时自动检查当前分类
- 在各业务模块达成条件时也可触发（如完成日记、打卡等）

解锁奖励：
1. 爱情树成长值（+20 普通 / +50 稀有）
2. 岛屿动态记录
