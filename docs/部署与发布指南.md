## 2. 后端部署（Docker Compose）

### 2.1 部署文件清单

项目根目录下 `deploy/` 文件夹中的文件：

```
deploy/
├── Dockerfile              # Spring Boot 容器构建
├── docker-compose.yml      # 服务编排（MySQL/Redis/MinIO/App/Nginx）
├── .env                    # 环境变量（数据库密码、JWT密钥、微信配置）
├── deploy.sh               # 一键部署脚本
├── init-db/
│   └── init.sql            # 数据库初始化（自动建库建表+种子数据）
└── nginx/
    └── default.conf        # Nginx 反向代理配置
```

### 2.2 配置 `.env` 文件

> ⚠️ **安全提醒**：部署前务必修改默认密码和密钥！

关键配置项说明：

| 变量 | 说明 | 建议 |
|------|------|------|
| `MYSQL_ROOT_PASSWORD` | MySQL root 密码 | 改为强密码 |
| `JWT_SECRET` | JWT 签名密钥 | 改为至少32位随机字符串 |
| `WECHAT_APP_ID` | 微信小程序 AppID | 从微信公众平台获取 |
| `WECHAT_APP_SECRET` | 微信小程序 AppSecret | 从微信公众平台获取 |
| `SERVER_HOST` | 服务器公网 IP | 当前为 `47.93.25.125`，有域名后改为域名 |

## 3. 服务器环境准备

### 3.1 安装 Docker

```bash
# 适用于 CentOS / Alibaba Cloud Linux
sudo yum install -y yum-utils
sudo yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
sudo yum install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin
sudo systemctl start docker
sudo systemctl enable docker

# 验证
docker --version
docker compose version
```

### 3.2 防火墙放行端口

```bash
# 如果使用 Alibaba Cloud 安全组，在控制台放行：
# - 80（HTTP）
# - 443（HTTPS，后续配置）
# - 8080（后端 API，测试用）
# - 9000（MinIO API）
# - 9001（MinIO 控制台，可仅对内网开放）

# 服务器内部防火墙（如有）
sudo firewall-cmd --add-port=80/tcp --permanent
sudo firewall-cmd --add-port=8080/tcp --permanent
sudo firewall-cmd --reload
```

> **生产建议**：在云服务商安全组中，3306（MySQL）、6379（Redis）**只对内网开放**，不对外暴露。

### 3.3 安装 Git（用于拉取代码）

```bash
sudo yum install -y git
```

## 4. 上传与部署

### 方式一：通过 Git 拉取代码（推荐）

```bash
# 在服务器上操作
mkdir -p /app
cd /app
git clone <你的代码仓库地址> lovers-plan
cd lovers-plan/deploy

# 编辑配置
vi .env    # 修改密码、密钥、微信配置

# 一键部署
chmod +x deploy.sh
./deploy.sh
```

### 方式二：本地打包上传

```bash
# 1. 本地打包代码（在项目根目录）
tar -czf lovers-plan.tar.gz \
  --exclude='node_modules' \
  --exclude='.git' \
  --exclude='.idea' \
  --exclude='target' \
  --exclude='__pycache__' \
  .

# 2. 上传到服务器
scp lovers-plan.tar.gz root@47.93.25.125:/app/

# 3. 服务器上解压并部署
ssh root@47.93.25.125
cd /app
tar -xzf lovers-plan.tar.gz
cd deploy
vi .env    # 修改密码、密钥
./deploy.sh
```

### 部署命令参考

```bash
cd /app/lovers-plan/deploy

./deploy.sh          # 首次部署（构建+启动）
./deploy.sh rebuild  # 代码更新后重新构建
./deploy.sh restart  # 仅重启
./deploy.sh stop     # 停止所有服务
./deploy.sh logs     # 查看实时日志
docker compose ps    # 查看容器状态
```

### 部署后验证

```bash
# 查看容器是否全部运行
docker compose ps

# 测试 API 是否可用
curl http://localhost:8080/api/test/ping

curl http://localhost/api/test/ping

# 查看日志
docker compose logs -f app
```

## 5. 配置 HTTPS 域名

> ⚠️ **微信小程序生产环境强制要求 HTTPS**，没有域名和 HTTPS 无法发布上线。

### 5.1 准备工作

1. **购买域名**（如 `loversplan.com`），完成 ICP 备案
2. **DNS 解析**：添加 A 记录指向 `47.93.25.125`
3. **确保端口 80/443 在安全组已放行**

### 5.2 申请 SSL 证书（Certbot）

```bash
# 进入 Nginx 容器或宿主机安装 certbot
# 推荐在宿主机操作
sudo yum install -y certbot python3-certbot-nginx

# 申请证书（会自动修改 Nginx 配置）
sudo certbot --nginx -d yourdomain.com

# 证书会自动续期（查看定时任务）
sudo crontab -l
```

### 5.3 启用 Nginx HTTPS 配置

编辑 `deploy/nginx/default.conf`，取消 HTTPS server 部分的注释，将 `yourdomain.com` 替换为真实域名：

```nginx
server {
    listen 443 ssl http2;
    server_name yourdomain.com;
    # ...
}
```

然后重启 Nginx：

```bash
docker restart lp-nginx
```

### 5.4 更新 API 地址

将小程序前端 `frontend/utils/api.js` 中的 BASE_URL 改为 HTTPS 域名地址：

```javascript
const BASE_URL = 'https://yourdomain.com/api';
```

### 5.5 配置微信小程序域名白名单

登录 [微信公众平台](https://mp.weixin.qq.com/) → **开发管理** → **开发设置** → **服务器域名**：

| 域名类型 | 填写内容 |
|---------|---------|
| request 合法域名 | `https://yourdomain.com` |
| uploadFile 合法域名 | `https://yourdomain.com` |
| downloadFile 合法域名 | `https://yourdomain.com` |

## 7. 小程序发布流程

### 7.1 发布前提

- ✅ 后端已部署到服务器并正常运行
- ✅ 已配置 HTTPS 域名（第 5 步）
- ✅ 微信公众平台已配置服务器域名白名单
- ✅ 小程序前端 BASE_URL 已改为 HTTPS 地址
- ✅ 小程序功能经过充分测试

### 7.2 发布步骤

```
微信公众平台 (mp.weixin.qq.com)
│
├── 1. 管理 → 版本管理 → 开发版本
│     在开发者工具中点击「上传」按钮
│     填写版本号（如 1.0.0）和备注
│
├── 2. 管理 → 版本管理 → 开发版本
│     找到刚上传的版本 → 点击「提交审核」
│
├── 3. 填写审核信息
│     ├── 功能页面：选择核心页面（首页、日记、任务等）
│     ├── 测试账号：提供测试用的微信账号（必须能正常登录）
│     └── 备注：说明核心功能流程
│
├── 4. 等待审核（通常 1-7 天）
│     审核结果会通过微信通知推送到小程序管理员微信
│
└── 5. 审核通过 → 点击「发布」
     可选择「全量发布」或「灰度发布」
```

### 7.3 常见审核拒绝原因及解决

| 拒绝原因 | 解决方案 |
|---------|---------|
| 功能不完整/无法体验 | 填写测试账号（需已绑定情侣），或在备注说明需要两个账号 |
| 缺少必要说明 | 在审核备注中详细描述功能流程和测试方法 |
| 涉及用户隐私但未说明 | 在 `app.json` 或备注中说明需要用户信息的原因 |
| 类目选择不当 | 选择合适的服务类目（社交 → 交友？工具 → 效率？） |

## 8. 让用户搜索到小程序

### 8.1 小程序的搜索入口

微信小程序的搜索来源主要是 **微信首页的搜索框**。用户搜索时，微信会展示相关的小程序。

### 8.2 提升搜索排名的关键因素

| 因素 | 操作建议 |
|------|---------|
| **名称** | 小程序名称要简洁易记，包含核心关键词 |
| **描述** | 在微信公众平台设置准确的功能描述（200字内） |
| **用户活跃度** | 日活跃用户越多，排名越高 |
| **搜索历史** | 已有用户搜索过这个小程序名字，排名会提升 |
| **服务评分** | 引导用户给好评 |

### 8.3 操作步骤

1. **设置小程序名称**：登录 [微信公众平台](https://mp.weixin.qq.com/) → **设置** → **基本设置** → **小程序名称**
   - 当前名称为「双人岛」— 情侣记录
   - 确保名称不超过20个字，不涉侵权

2. **设置搜索关键词**：公众平台 → **功能** → **搜索** → **关键词配置**
   - 填写：双人岛、情侣、恋爱、情侣日记、情侣任务、纪念日、时光胶囊
   - 最多可配置10个关键词

3. **优化功能描述**：公众平台 → **设置** → **基本设置** → **功能描述**
   - 示例："双人岛是一款专为情侣设计的私密成长空间。写恋爱日记、设置纪念日、打卡情侣任务、绘制爱情树…一起记录属于你们的爱情故事。"

4. **设置服务类目**：确保类目选择准确，影响搜索曝光

### 8.4 扩展推广渠道

除了搜索，还可以通过以下方式获取用户：

- **二维码海报**：生成小程序码，分享到朋友圈/群聊
- **公众号关联**：如果有公众号，关联到小程序菜单
- **线下物料**：打印小程序码放在门店/活动物料上
- **微信广告**：投放微信搜一搜广告（需要预算）

### 8.5 发布后的检查清单

- [ ] 小程序可以正常打开和登录
- [ ] 情侣绑定流程正常
- [ ] 写日记、上传图片功能正常
- [ ] 任务打卡功能正常
- [ ] 纪念日提醒功能正常
- [ ] 爱情树成长值正常累计
- [ ] 时光胶囊可以创建和开启
- [ ] 每日问答可以正常参与
- [ ] 成就可以正常解锁
- [ ] 搜索小程序名称能正常找到
